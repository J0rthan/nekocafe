# D4-1 测试计划与策略

## 1. 测试目标与质量目标

### 1.1 质量目标（基于 ISO/IEC 25010）
| 质量特性 | 子特性 | 目标值 | 度量方法 |
|---------|--------|--------|---------|
| 功能适用性 | 功能正确性 | 所有API端点返回正确响应 | 自动化测试通过率 ≥ 95% |
| 可靠性 | 容错性 | 服务优雅处理异常输入 | 异常输入返回4xx，非5xx |
| 性能效率 | 时间行为 | P95响应时间 < 2s | k6 压测 |
| 安全性 | 机密性 | 无高危漏洞 | Bandit SAST + OWASP ZAP |
| 可维护性 | 可测试性 | 代码覆盖率 ≥ 70% | pytest-cov |

### 1.2 量化指标
| 指标 | 目标 | 实际达成 |
|------|------|---------|
| 行覆盖率 (Line) | ≥ 70% | 94% (member) / 93% (reservation) |
| 分支覆盖率 (Branch) | ≥ 60% | 91% (member) / 90% (reservation) |
| 变异分数 (Mutation Score) | ≥ 60% | 66.7% |
| 缺陷密度 | ≤ 5/KLOC | TBD |
| 缺陷逃逸率 | ≤ 10% | TBD |

| 安全测试 | Baseline Scan | OWASP ZAP + Bandit | < 5min |

## 3. 测试数据管理策略

### 3.1 测试数据生成
- **单元测试**: 使用 hypothesis 自动生成随机测试数据
- **集成测试**: 使用固定 fixtures + 随机化 customer_id
- **E2E 测试**: 使用独立的测试数据集，避免污染生产数据

### 3.2 敏感数据脱敏
- **手机号**: `mask_phone()` 函数 → `138****8001`
- **邮箱**: 仅存储测试域邮箱 `@nekocafe.test`
- **会员ID**: 使用 UUID，不包含PII
- **密码**: 不存储明文密码（本系统使用无密码认证）

### 3.3 数据清理策略
- 每个测试套件运行后自动清理 `_fake_db`
- 集成测试后执行 `docker compose down -v` 清理卷
- CI 流水线使用临时 PostgreSQL 实例

## 4. 测试自动化与 CI 集成

### 4.1 本地执行
```
make test          # 运行所有单元测试 + 覆盖率
make lint          # 代码检查 (flake8 + black)
make scan          # 安全扫描 (Bandit)
```

### 4.2 CI 流水线触发
- **PR → main/develop**: 自动运行 lint → unit-test → sast → build → integration-test
- **Push → main**: 加上 push-image
- **E2E 测试**: 作为 CD 准入门槛，在 staging 部署后自动执行

### 4.3 质量门禁
- 代码覆盖率 < 70% → 构建失败
- Bandit 发现 HIGH 级别问题 → 告警
- Trivy 发现 CRITICAL 漏洞 → 构建失败
- E2E 测试失败 → 阻止部署到生产

## 5. 缺陷管理流程

### 5.1 缺陷分类（严重度 × 优先级）
| 优先级\严重度 | Critical | Major | Minor | Trivial |
|-------------|----------|-------|-------|---------|
| P0 (立即)    | 🔴       | 🔴    | 🟡    | -       |
| P1 (高)      | 🔴       | 🟡    | 🟢    | -       |
| P2 (中)      | 🟡       | 🟢    | 🟢    | 🟢      |
| P3 (低)      | 🟢       | 🟢    | ⚪    | ⚪      |

```
发现 → 录入 GitHub Issue → 分析 → 修复 → PR → 代码审查 → 合并 → 验证关闭
```

## 6. 风险评估
| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 测试环境不稳定 | 中 | 中 | Docker Compose 健康检查 + 重试机制 |
| 假阳性测试 | 低 | 中 | 定期审查测试质量 + 变异测试 |
| 覆盖率虚假达标 | 高 | 低 | 变异测试验证测试质量 |
| 性能回归 | 中 | 低 | CI 中加入性能基线对比 |
