# D4-2 测试用例设计文档

## 概述

基于 ISO/IEC 25010 质量模型，针对 NekoCafé 平台的会员服务和预约服务设计全面的测试用例。每条测试用例可追溯到实验一的需求 ID。

---

## 1. 会员服务测试用例

### 1.1 单元测试

| ID | 需求追溯 | 测试名称 | 前置条件 | 输入 | 预期输出 | 类型 |
|----|---------|---------|---------|------|---------|------|

| UT-M-002 | REQ-M-02 | 创建新会员 | 无 | POST /members {name,phone,...} | 201, tier=bronze, points=0 | 正常 |
| UT-M-003 | REQ-M-02 | 创建会员-无效手机号 | 无 | POST /members {phone:"12345"} | 422 | 异常 |

| UT-M-005 | REQ-M-03 | 查询不存在会员 | 无 | GET /members/nonexistent | 404 | 异常 |
| UT-M-006 | REQ-M-04 | 更新会员信息 | 会员存在 | PATCH /members/{id} | 200, 更新后信息 | 正常 |
| UT-M-007 | REQ-M-05 | 查询会员列表 | 有会员 | GET /members | 200, 数组 | 正常 |

### 1.2 基于属性测试 (PBT)

| ID | 需求追溯 | 被测函数 | 属性/不变量 | 策略 |
|----|---------|---------|------------|------|
| PBT-M-01 | REQ-M-07 | calculate_points | 输出始终在 [10, 5000] | 随机 guest_count, duration, is_first_visit |
| PBT-M-02 | REQ-M-07 | calculate_points | 首次访问积分 ≥ 非首次 | 相同参数，仅 is_first_visit 不同 |
| PBT-M-03 | REQ-M-07 | calculate_points | 更多客人 = 更多积分 | g1 < g2 → p1 ≤ p2 |
| PBT-M-04 | REQ-M-08 | determine_tier | 积分单调 → 等级单调 | 随机 total_points |
| PBT-M-05 | REQ-M-08 | determine_tier | 更多积分不降低等级 | p1 ≤ p1+delta → tier不变或升级 |
| PBT-M-06 | REQ-M-08 | should_upgrade | 新等级 ≥ 当前等级 | 随机 points + current_tier |
| PBT-M-07 | REQ-M-09 | mask_phone | 格式: 3+"****"+4 | 随机手机号 |
| PBT-M-08 | REQ-M-09 | mask_phone | 脱敏后不等于原值 | 随机手机号 |
| PBT-M-09 | REQ-M-10 | validate_display_name | 字母数字名有效 | 随机字母数字名 |
| PBT-M-10 | REQ-M-10 | validate_display_name | 过长/全数字 = 无效 | 边界条件 |
| PBT-M-11 | - | calculate_age | 年龄 ≥ 0 | 随机天数 |
| PBT-M-12 | - | calculate_age | 一年后年龄+1 | 同生日不同参考日期 |

## 2. 预约服务测试用例

### 2.1 单元测试

| ID | 需求追溯 | 测试名称 | 输入/条件 | 预期输出 | 类型 |

| UT-R-002 | REQ-R-01 | 按门店过滤桌位 | GET /tables?store_id=store-002 | 仅 store-002 | 正常 |

### 2.2 基于属性测试 (PBT)

| ID | 需求追溯 | 被测函数 | 属性 | 策略 |
|----|---------|---------|------|------|
| PBT-R-01 | REQ-R-01 | allocate_table | 分配桌位容量 ≥ 人数 | 随机 guest_count + 桌位列表 |
| PBT-R-02 | REQ-R-01 | allocate_table | 超量返回 None | guest_count=100, 小桌位 |
| PBT-R-03 | REQ-R-01 | allocate_table | window 偏好被尊重 | window vs center 桌位 |
| PBT-R-04 | REQ-R-02 | detect_conflict | 时间段重叠检测冲突 | 随机 overlap_minutes |
| PBT-R-05 | REQ-R-02 | detect_conflict | 非重叠无冲突 | gap > buffer |
| PBT-R-06 | REQ-R-02 | detect_conflict | 已取消预约忽略 | 随机 status |
| PBT-R-07 | REQ-R-04 | can_cancel | 阈值内/外正确判断 | 随机 hours_ahead + tier |
| PBT-R-08 | REQ-R-04 | can_cancel | platinum 随时可取消 | always true |
| PBT-R-09 | REQ-R-06 | calculate_price | 价格 > 0 | 随机所有参数 |
| PBT-R-10 | REQ-R-06 | calculate_price | 高峰期更贵 | peak vs off-peak |
| PBT-R-11 | REQ-R-06 | calculate_price | 更高等级更低价 | 同参数不同 tier |
| PBT-R-12 | REQ-R-06 | calculate_price | 附加服务加价 | 加与不加 extra_services |
